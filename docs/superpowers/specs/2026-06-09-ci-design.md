# CI GitHub Actions — Design

## Goal

Exécuter typecheck + tests Jest à chaque push sur `main`. Badge rouge = régression détectée.

## Architecture

Un seul fichier : `.github/workflows/ci.yml` à la racine du repo (pas dans `app/`).

Un job `ci` sur `ubuntu-latest`, Node 20 LTS, working directory `app/`.

## Workflow

**Trigger :** `push` sur `main` uniquement.

**Steps :**
1. `actions/checkout@v4`
2. `actions/setup-node@v4` — Node 20, `node-version: '20'`
3. `npm ci` — install déterministe depuis `package-lock.json`
4. `npm run typecheck` — `tsc --noEmit`
5. `npm test -- --no-coverage` — Jest, pas de rapport coverage

## Fichier à créer

| Action | Fichier |
|--------|---------|
| Créer | `.github/workflows/ci.yml` |

## Décisions

- Pas de cache — overhead de config non justifié pour projet solo.
- Pas de PR trigger — push direct sur main, pas de workflow PR.
- `--no-coverage` — coverage report ralentit sans valeur ajoutée en CI.
- Node 20 LTS — compatible Expo SDK 54 / RN 0.81.
